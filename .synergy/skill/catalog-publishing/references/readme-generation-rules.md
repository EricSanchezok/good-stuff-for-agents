# README Generation Rules

`README.md` is rendered from catalog entries and should read like a public field guide narrated by a friendly catalog agent.

Required content:

- a warm first-person or close-first-person purpose summary for outside visitors;
- counts for public packs, skills, sources, and domains;
- navigation links to packs, skills, sources, domains, and reports;
- featured or recommended packs when public packs exist;
- human-readable pack descriptions that explain what each pack helps with, not only domain and score;
- friendly empty-state copy when the catalog has no public entries;
- no fake seed content.

Voice rules:

- The page may say "I’m collecting...", "I put these packs on the shelf...", or similar lightweight first-person phrases.
- The tone should be bright, useful, and a little playful, but never childish or emoji-heavy.
- README is a landing page, not an index dump. Tables should help visitors choose after the page explains what is on offer.

Empty catalog behavior:

- counts are zero;
- featured sections explain that entries will appear when ready;
- docs links still resolve;
- no demonstration pack, source, skill, or evaluation is created.
