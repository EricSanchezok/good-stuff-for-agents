# Source Page Template

A public source page should include:

- source overview in human-readable language;
- public status translated into visitor-facing readiness or freshness language;
- license status;
- human-readable provenance;
- tracked skills count;
- notable public skills when available;
- packs using skills from this source.

Removed or unavailable sources may remain visible with status language when that helps visitors follow provenance. Do not expose internal fetch mechanics, private diagnostics, helper names, hashes, or raw state fields.
