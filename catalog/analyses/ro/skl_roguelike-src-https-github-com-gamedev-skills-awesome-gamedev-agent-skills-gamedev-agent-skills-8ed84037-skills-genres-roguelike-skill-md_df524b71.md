# Roguelike

## Core Purpose
Guides agents through roguelike game development — procedural dungeon generation, permadeath and meta-progression, turn-based movement, line-of-sight and FOV, and procedural loot/stat systems.

## Trigger Semantics
Load when an agent builds a roguelike or roguelite game, implements procedural level generation, designs permadeath with meta-progression, or creates procedural loot and character builds.

## Capability Breakdown
Covers procedural dungeon generation (BSP, random walk, cellular automata), field-of-view and line-of-sight algorithms (raycasting, shadowcasting), turn-based movement and action queuing, permadeath and unlock/unlock meta-progression, procedural loot and affix systems, and character build variety through procedural stats.

## Workflow Role
Implementation — applied when building roguelike/roguelite core gameplay loops.

## Inputs / Outputs
Inputs: game design specs for dungeon generation and progression. Outputs: dungeon generator, FOV system, meta-progression manager.

## Tool and Permission Profile
Standard code editing.

## Compatibility Notes
Complements procedural-gen for dungeon generation and save-systems for permadeath and meta-progression save data.
