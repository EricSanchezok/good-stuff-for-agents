# Copywriting Frameworks

A reference guide covering seven copywriting frameworks (AIDA, PAS, BAB, 4Ps, QUEST, FAB, StoryBrand), eight headline formulas, CTA writing rules, and a framework-selection table. It is purely instructional — no extraction, no generation workflow, no validation. It teaches an agent the shape of persuasive copy and when to use which structure.

## Why it matters

This is a competent, above-average copywriting cheat sheet. What elevates it from generic is the framework-selection guide — a lookup table mapping specific situations (short ad copy, landing page hero, email sequence, cold outreach) to the best framework with a one-line rationale. Most copywriting reference skills dump the frameworks and leave the agent to guess which one fits. The selection table turns a reference into a decision tool.

The CTA anti-patterns section is the second genuinely useful piece. Banning "Submit," "Click Here," and context-free "Sign Up" gives an agent concrete editing rules rather than vague "write better CTAs" advice. This is checklist-level actionable, not aspirational.

The examples are well-chosen — each framework gets a worked example in a realistic SaaS/marketing-tech context, so the agent sees the template instantiated, not just described. The StoryBrand section correctly positions the customer as the hero (a common mistake is casting the brand as hero), and the FAB section demonstrates the feature→advantage→benefit chain rather than just defining it.

The weakness: this is reference material, not a skill. There is no workflow — no instruction on how to gather product context, how to interview the user for value props, how to iterate with feedback, how to handle edge cases like compliance-heavy industries. An agent loading this skill gets templates, not a process.

## Where it helps, where it hurts

**Best-case scenario:** You have solid product context already — you know the pain points, the value props, the audience. You need to write a landing page hero, a cold email sequence, and three Google Ads. Loading this skill gives you framework selection (4Ps for hero, BAB for cold email, PAS for ads), structure templates, headline formulas, and CTA patterns all in one reference. The output is structurally sound copy that follows proven persuasion patterns.

**Worst-case scenario:** The agent has minimal product context and the user says "write me some good copy for my SaaS." The agent loads this skill, picks AIDA because it's first, fills in the template with generic SaaS tropes ("transform your workflow," "unlock revenue"), and produces copy that reads like every AI-written landing page. The frameworks don't prevent bad inputs — they structure them. Worse, the templates can create an illusion of quality: copy that has the right shape but no substance. The CTA anti-patterns won't save you when "Start Saving Today" is attached to a product the agent doesn't understand.

## What it quietly assumes

The biggest unstated assumption is that the agent already knows enough about the product, market, and audience to fill the templates with specific, credible content. The frameworks provide structure, not substance. AIDA says "hook with a bold claim" but doesn't help the agent discover what claim is defensible. PAS says "name the pain" but doesn't help surface customer research. This is fine if the agent arrives with context from a brand-dna or customer-research skill — but this skill doesn't declare that dependency.

Other assumptions: the copy will be read by a Western, English-speaking audience (the persuasion patterns are culturally specific); conversion is the primary goal (no support for awareness, retention, or brand-building copy); the product is differentiated enough that FAB and comparison formulas work (commodity products fill these templates with meaningless differentiators); the agent can judge when to break the rules (the CTA anti-patterns are presented as absolute, but "Learn More" is genuinely the right CTA for some awareness-stage contexts).

The headline formulas assume the agent has credible numbers and social proof to cite. "How Glossier Grew 600% With Email Alone" only works if you can back it up.

## What could go wrong

The risks here are quality risks, not operational risks. The worst realistic outcome is producing copy that passes a template check but fails a human reader — formulaic, generic, or actively misleading because the agent filled in specifics it shouldn't have. An agent that doesn't know the difference between a real differentiator and a cliché will produce clichés in AIDA format.

There's also a compliance risk in regulated industries: the frameworks encourage bold claims ("Your email campaigns are leaving $80K on the table") without any guardrails for substantiation. Drawn into finance, health, or legal copy, this skill could produce claims that create liability.

No tool permissions beyond text generation are needed. The user should review output, especially claims with specific numbers.

## Bottom line

Pick it as a quick-reference cheat sheet when you already have product context and just need structural guidance. The framework-selection table and CTA anti-patterns earn their keep. Skip it if you need a copywriting process rather than a template library, or if your product context is thin — this skill won't rescue bad inputs. Biggest risk is template-shaped generic copy; biggest benefit is the selection table that prevents framework-choice paralysis. In a tight 100-skill catalog: yes, if kept honest about being a reference, not a workflow.
