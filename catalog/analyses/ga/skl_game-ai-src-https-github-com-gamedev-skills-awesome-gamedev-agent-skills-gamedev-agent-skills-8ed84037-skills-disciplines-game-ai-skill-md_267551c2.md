# Game Ai

## Core Purpose
Guides agents through implementing game AI — behavior trees, finite state machines, pathfinding (A*, navigation meshes), utility AI, and enemy/bot behavior design for various game genres.

## Trigger Semantics
Load when an agent implements NPC behavior in a game, designs enemy AI patterns, sets up pathfinding agents, creates behavior trees or finite state machines, or tunes AI difficulty.

## Capability Breakdown
Covers AI architecture selection (FSM, behavior trees, utility AI, GOAP), behavior tree implementation (selectors, sequences, decorators), A* pathfinding and navigation mesh setup, state machine design patterns, sensory systems (vision, hearing), and difficulty balancing through AI parameters.

## Workflow Role
Implementation — applied during gameplay programming when creating non-player character behavior and decision-making systems.

## Inputs / Outputs
Inputs: NPC behavior requirements, game world data (nav mesh, obstacle data). Outputs: AI controller implementation, behavior tree definitions, pathfinding agents.

## Tool and Permission Profile
Standard code editing, game engine AI tooling (behavior tree editors).

## Compatibility Notes
Complements godot-nodes-scenes, unreal-behavior-trees for engine-specific AI implementation. Works with game-feel for AI-driven game feel.

## Conflict Notes
No conflicts. AI design principles are engine-agnostic and complementary to engine-specific AI skills.

## Dedupe Notes
Comprehensive game AI design skill — distinct from engine-specific AI skills which focus on specific engine APIs. No duplicate.

## Evaluation Hooks
Verify that the skill covers both reactive and deliberative AI approaches and that behavior tree patterns match standard game AI conventions.

medium — based on source metadata and game AI design knowledge. Full content review recommended.
