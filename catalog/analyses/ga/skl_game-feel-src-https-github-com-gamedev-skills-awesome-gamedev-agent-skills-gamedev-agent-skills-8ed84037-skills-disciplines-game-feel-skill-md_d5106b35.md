# Game Feel

## Core Purpose
Guides agents through designing and implementing game feel — the juice and polish elements that make games satisfying: screen shakes, particle effects, hit feedback, animation blending, timing curves, and responsive controls.

## Trigger Semantics
Load when an agent polishes game interactions, adds feedback effects (juice), tunes control responsiveness, implements hit confirmation (hit-stop, screen shake), or improves the moment-to-moment satisfaction of gameplay.

## Capability Breakdown
Covers feedback effects (screen shake, hit-flash, hit-stop), movement and control smoothing (acceleration curves, input buffering), particle and VFX triggers for actions, tweening and easing for UI/game elements, audio feedback synchronization, and controller rumble implementation.

## Workflow Role
Implementation/Polish — applied throughout development but especially during polish phases to make interactions feel responsive and satisfying.

## Inputs / Outputs
Inputs: game interaction events, desired feel characteristics. Outputs: feedback effect implementations, tuned parameters, polish code.

## Tool and Permission Profile
Standard code editing, game engine integration, animation tools.

## Compatibility Notes
Complements camera-systems for camera-based game feel, audio-design for audio feedback, and shader-development for visual feedback effects.

## Conflict Notes
No conflicts. Game feel is additive polish that builds upon existing systems.

## Dedupe Notes
Unique in the catalog — no other skill covers game feel/juice design comprehensively.

## Evaluation Hooks
Verify that feel effects are performance-conscious (lightweight) and that the skill covers both 2D and 3D feel patterns.

## Evidence and Confidence
medium — based on source metadata and game feel design knowledge. Full content review recommended.
