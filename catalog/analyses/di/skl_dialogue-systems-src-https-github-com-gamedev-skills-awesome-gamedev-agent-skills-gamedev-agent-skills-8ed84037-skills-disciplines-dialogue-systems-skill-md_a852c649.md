# Dialogue Systems

## Core Purpose
Guides agents through implementing game dialogue systems — branching dialogue trees, node-based dialogue editors, localization support, character voice assignment, and integration with quest and narrative systems.

## Trigger Semantics
Load when an agent builds a dialogue system for a game, implements branching conversation logic, integrates Ink/YarnSpinner dialogue scripting, or sets up dialogue localization pipelines.

## Capability Breakdown
Covers dialogue data structures (nodes, links, conditions), branching logic and condition evaluation, dialogue scripting languages (Ink, YarnSpinner), UI presentation (typewriter text, choice menus), localization string tables for dialogue, voice-over line management, and integration with quest/narrative state tracking.

## Workflow Role
Implementation — applied during narrative game development when creating the conversation and interaction systems.

## Inputs / Outputs
Inputs: dialogue script content, branching rules, UI design specs. Outputs: dialogue system implementation, runtime data structures, localization pipeline.

## Tool and Permission Profile
Standard code editing, dialogue scripting tools (Inky, YarnSpinner editor).

## Compatibility Notes
Complements game-ui-ux for dialogue UI presentation. Works with genre skills for narrative-driven game types (rpg, visual-novel).

## Conflict Notes
No conflicts. Narrative dialogue systems are independent of other gameplay systems.

## Dedupe Notes
Unique in the catalog — no other skill covers game dialogue systems. Related to but distinct from generic text/UI skills.

## Evaluation Hooks
Verify that the skill covers both simple (linear) and complex (branching with conditions) dialogue and that localization patterns handle character encoding and text expansion.

## Evidence and Confidence
medium — based on source metadata and game dialogue system knowledge. Full content review recommended.
