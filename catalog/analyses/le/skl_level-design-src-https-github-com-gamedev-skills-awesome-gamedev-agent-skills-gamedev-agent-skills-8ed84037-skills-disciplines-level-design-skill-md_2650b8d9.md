# Level Design

## Core Purpose
Guides agents through game level design principles and implementation — level layout, pacing, player guidance, encounter design, blockout/prototyping, and level scripting for various game genres.

## Trigger Semantics
Load when an agent designs game levels, blocks out level geometry, places enemies and objectives, implements level scripting events, or tunes level pacing and difficulty curves.

## Capability Breakdown
Covers level layout principles (flow, sightlines, choke points), blockout and greyboxing workflow, player guidance techniques (lighting, signage, landmarks), encounter and puzzle placement, level scripting triggers and sequences, and difficulty curve tuning.

## Workflow Role
Design/Implementation — applied during game world creation, from initial blockout to final level polish.

## Inputs / Outputs
Inputs: game genre, target difficulty, level narrative context. Outputs: level layouts, encounter placements, scripted sequences, blockout geometry.

## Tool and Permission Profile
Standard game editor tools (Godot, Unity, Unreal), script editing for level events.

## Compatibility Notes
Complements procedural-gen for procedural level content and game-ai for enemy placement considerations. Pairs with platformer, rpg, or fps-shooter genre skills.

## Conflict Notes
No conflicts. Level design is a creative/structural discipline that complements engine and genre skills.

## Dedupe Notes
Unique in the catalog — no other skill covers game level design comprehensively.

## Evaluation Hooks
Verify that the skill covers both 2D and 3D level design patterns and that pacing guidance addresses different genre requirements.

## Evidence and Confidence
medium — based on source metadata and game level design knowledge. Full content review recommended.
