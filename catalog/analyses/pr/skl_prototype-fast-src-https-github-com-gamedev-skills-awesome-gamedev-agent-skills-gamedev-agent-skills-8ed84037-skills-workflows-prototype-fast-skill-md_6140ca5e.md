# Prototype Fast

## Core Purpose
Guides agents through fast game prototyping — identifying core mechanics, building minimal playable prototypes, using placeholder assets, playtesting early, and iterating rapidly on game feel.

## Trigger Semantics
Load when an agent prototypes a game idea quickly, validates core mechanics, or builds a minimum playable version for early testing.

## Capability Breakdown
Covers core mechanic extraction (identifying the fun), minimum playable prototype (vertical slice), placeholder asset pipelines (colored shapes, free sound effects), rapid iteration loops (build, test, adjust), and prototype-specific project organization for speed.

## Workflow Role
Planning/Implementation — applied in early stages of game development before full production.

## Inputs / Outputs
Inputs: game concept, core mechanic description. Outputs: playable prototype, validated mechanic feedback.

## Tool and Permission Profile
Standard game engine tools.

## Compatibility Notes
Complements game-jam for jam prototyping and game-playtesting for early feedback integration.

## Conflict Notes
No conflicts.

## Dedupe Notes
Unique in the catalog — no other rapid prototyping skill.

## Evaluation Hooks
Verify the skill covers how to identify the core fun loop and build to validate it.
